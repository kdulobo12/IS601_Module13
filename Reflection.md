# Reflection

This assignment helped me understand how authentication works in a full-stack application. I implemented JWT-based registration and login routes using FastAPI. The registration route accepts user information, validates it using Pydantic schemas, checks if the user already exists, hashes the password, and stores the user in the PostgreSQL database. The login route verifies the user credentials by checking the hashed password and returns a JWT token when the login is successful. One important part of this assignment was learning how password security works. Instead of storing plain text passwords, I used password hashing so that the original password is not saved directly in the database. This made the authentication system more secure and showed me why hashing is important in real applications.

I also created front-end pages for registration and login using HTML, CSS, and JavaScript. The registration page checks that the email format is valid and that the password meets the minimum length requirement. The login page allows the user to enter their credentials and stores the JWT token in localStorage after a successful login. This helped me understand how the front-end communicates with the backend API. Another major part of this project was writing Playwright end-to-end tests. I created positive tests for successful registration and login, and negative tests for invalid inputs such as a short password and wrong login credentials. These tests helped confirm that the application works correctly from the user’s perspective, not just from the backend.

I also maintained the Docker-based setup and GitHub Actions CI/CD pipeline. The pipeline starts the database and server, runs the automated tests, and builds the Docker image. This helped me understand how automated testing and deployment are used in real software development workflows. Some challenges I faced included connecting the FastAPI backend with PostgreSQL inside Docker, making sure the front-end correctly displayed success and error messages, and writing Playwright tests that waited for the correct UI changes. Debugging these issues helped me better understand Docker networking, API responses, and automated browser testing.

Overall, this assignment gave me more confidence in building secure full-stack applications. I learned how JWT authentication, password hashing, front-end validation, Playwright testing, Docker, and GitHub Actions all work together in a complete development pipeline.
